🚀 博客技术架构全解析 ​

本博客系统采用了一套极具性价比且性能卓越的现代化技术栈，实现了内容管理自动化、鉴权中转无缝化以及前端渲染极致化。

🛠️ 核心框架与工具链 ​

1. VitePress (静态站点生成器) ​

  • 定位：由 Vite 驱动的 Vue 极简静态网站生成器。
  • 特性：
    • 极致性能：基于 Vite 构建，拥有极快的热更新速度和首屏加载性能。
    • Markdown 增强：支持在 Markdown 中直接编写 Vue 组件。
    • 现代渲染：生成静态 HTML 的同时，作为单页应用（SPA）在浏览器中运行。

2. Decap CMS (内容管理系统) ​

  • 定位：一个基于 Git 的开源内容管理系统。
  • 特性：
    • Git-based：所有通过后台发布的文章都会直接转化为仓库中的 commit，无需传统数据库。
    • 配置驱动：通过 config.yml 定义字段结构，支持复杂的嵌套列表和文件管理。
    • 实时预览：在浏览器端提供所见即所得的编辑体验。

3. Cloudflare Worker (鉴权中转层) ​

  • 定位：轻量级边缘计算平台。
  • 特性：
    • Serverless：作为 GitHub OAuth 鉴权的中间件，处理复杂的 Token 交换逻辑。
    • 零成本：利用免费额度实现稳定、安全的后台登录功能。
    • 全球加速：在靠近用户的边缘节点运行，响应速度极快。

4. GitHub Actions (自动化部署) ​

  • 定位：CI/CD 自动化流水线。
  • 特性：
    • 全自动流程：每当仓库有新的 push（来自本地或 CMS 后台），自动触发构建脚本。
    • 静态托管：自动将编译后的静态文件部署至 GitHub Pages 供全球访问。

🏗️ 系统架构图解 ​

整个博客的运行逻辑可以拆解为以下三个核心链路：

  1. 内容录入链： Decap CMS (浏览器) -> Cloudflare Worker (鉴权) -> GitHub Repo (源码存储)
  2. 构建部署链： GitHub Push -> GitHub Actions (Vite 编译) -> GitHub Pages (静态发布)
  3. 用户访问链： 用户浏览器 -> GitHub Pages (CDN) -> 加载渲染好的 VitePress 页面

🌟 为什么选择这套架构？ ​

对于追求极致体验的开发者来说，这套方案拥有以下不可比拟的优势：

  • 零配置维护：完全脱离了传统服务器（Nginx/Node.js/Database）的繁琐运维。
  • 永久免费：利用 GitHub 和 Cloudflare 的免费额度，实现了零成本运行。
  • 高性能表现：借助本地高性能开发环境配合 Vite 的毫秒级响应，确保开发与访问流畅无比。
  • 数据安全：所有文章即代码，拥有完善的版本控制历史，无需担心数据库损坏。
