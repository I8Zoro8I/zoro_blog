PyTorch 实战外传 —— 计算机视觉识别烟叶种类与物联网控制 ​

在智慧农业与工业自动化分拣的浪潮下，利用计算机视觉技术代替人工进行农作物分选已成为大势所趋。本文将分享一个完整的工业级实战案例：基于经典卷积神经网络 VGG19，对高达到 25GB 的烟叶数据集进行工业级全链路深耕，实现对“上部叶”、“中部叶”和“下部叶”的精准识别与自动化硬件分拣。 项目不仅打通了从「数据打标」到「模型训练」的算法闭环，更向下延伸至硬件底层，结合树莓派 GPIO 硬件编程与 OpenCV 摄像头实时流，构筑了“上机智能识别，线下硬件舵机精准拨档分拣”的物联网控制系统。

📌 项目核心亮点 ​

  • 📦 海量工业级数据集：项目坐拥高达 25GB 的真实烟叶图像数据，高数据容量保障了模型的泛化上限。(国庆放假七天兢兢业业拍了五天┭┮﹏┭┮)
  • 🏷️ 7:2:1 黄金数据集划分：严谨的数据集工程，规避过拟合，真实反馈验证集与测试集的业务表现。
  • 🎯 精准的损失函数与优化器选型：采用 CrossEntropyLoss 交叉熵损失函数配合 SGD 随机梯度下降，确保多分类问题的高效收敛。
  • ⚙️ 边缘硬件协同（IoT）：集成树莓派 GPIO 接口，利用 PWM 平滑移动滤波算法 控制舵机物理摆动，拒绝分拣臂机械抖动。

📂 第一步：数据工程 —— 打标与 7:2:1 数据集划分 ​

面对 25GB 的海量鲜烟叶原始图片，不能直接一股脑喂给模型。标准的工业级流水线第一步必然是标准化打标与集合切分。

1. 标签映射关系 ​

根据生产标准(其实远远不止这些分类,这里只做大类区分)，我们创建了 3 个分类文件夹（对应 ImageFolder 的自动标签映射）：

  • 0 — bottom（下部叶）：颜色偏淡黄，叶片较薄。
  • 1 — mid（中部叶）：金黄色，油分充足，品质最高。
  • 2 — top（上部叶）：颜色深黄绿，身骨较重，叶片肥厚。

2. 7:2:1 动态切分策略 ​

由于 25GB 的数据量直接进行全盘手动复制容易导致磁盘 I/O 崩溃，我们在清洗时通过脚本按照 70% 训练集 (Train)、20% 验证集 (Val)、10% 测试集 (Test) 进行了无放回随机抽样划分：

  • 训练集（70%）：供模型反复前向传播与反向擦除，学习烟叶的脉络与色泽特征。
  • 验证集（20%）：在每轮（Epoch）训练结束后，作为无梯度调优的测试卷，用来挑选并保存正确率最高的模型权重（防止过拟合）。
  • 测试集（10%）：完全隔离于训练流程之外，最终用于模拟盲测，跑出真实的工业级分拣成功率。

🏗️ 第二步：核心模型构建与训练模块 (train.py) ​

在训练阶段，我们选用了大容量的经典网络 VGG19。为了完美适配我们的 3 分类业务，通过 nn.Linear(4096, 3) 强行重构了全连接层的输出。

📝 选型思考：为什么选择 CrossEntropyLoss 和 SGD？ ​

  • 损失函数（CrossEntropyLoss）：烟叶分类属于互斥多分类问题。CrossEntropyLoss 将 LogSoftmax 和 NLLLoss（负对数似然损失）合二为一。它能计算预测出的各部位概率分布与真实独热编码（One-Hot）之间的拉普拉斯距离。当模型对正确品类信心不足时，会施加极大的惩罚，迫使模型快速修正权重。
  • 优化器（SGD）：在处理 25GB 这样的大规模数据集时，类似 Adam 这样的自适应学习率优化器容易在训练中后期陷入局部马鞍点，导致泛化性能（Generalization）变差。而配合了特定学习率（1e-3）的 SGD（随机梯度下降） 能够保证梯度平滑下降，训练出的特征边缘更加扎实，非常适合工业界对图像纹理和色彩的收敛要求。
python
# -*- encoding: UTF-8 -*-
# @ModuleName: train
# @Function: 7:2:1 烟叶多分类工业训练核心脚本
# @Author : Zoro

import torch
import torchvision
from torch import nn
from torchvision import transforms, datasets
from torch.utils.data import DataLoader

# 1. 定义训练的计算设备 (优先使用 GPU 显卡加速)
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
print(f"📡 当前训练加速硬件状态: {device}")

# 2. 数据工程路径配置
train_path = "./dataset2/train"  # 占比 70% 的训练集
test_path = "./dataset2/test"    # 占比 10% 的独立盲测试集
# 提示：占比 20% 的验证集通常在每轮 Epoch 结束后做推理评估，这里复用评估逻辑

# 3. 数据增强与高阶工业图像预处理
transforms_pipeline = transforms.Compose([
    transforms.Resize(256),         # 保持长宽比，将短边缩放至 256,降低计算负载
    transforms.CenterCrop(224),     # 严丝合缝裁剪出核心区域 3*224*224 矩阵,聚焦烟叶主体，统一输入矩阵维度。
    transforms.ColorJitter(contrast=0.5), # 注入对比度变换，对抗工业环境的光照抖动
    transforms.ToTensor()           # 归一化至 [0, 1] 并转换为 PyTorch Tensor 数学归一化，将图像像素升级为“高维数字张量”。
])

# 4. 载入数据集
dataset_train = datasets.ImageFolder(train_path, transform=transforms_pipeline)
dataset_test = datasets.ImageFolder(test_path, transform=transforms_pipeline)

train_data_size = len(dataset_train)
test_data_size = len(dataset_test)
print(f"📊 成功载入数据！训练集样本数: {train_data_size} | 测试集样本数: {test_data_size}")

# 5. 构建高并发 DataLoader
train_data_loader = DataLoader(dataset_train, batch_size=15, shuffle=True, num_workers=4, pin_memory=True)
test_data_loader = DataLoader(dataset_test, batch_size=15, shuffle=True, num_workers=4, pin_memory=True)

# 6. 改造 VGG19 骨干网络
model = torchvision.models.vgg19(weights=None) # 从头训练 25G 的烟叶专项特征
model.classifier[6] = nn.Linear(4096, 3)       # 强行粉碎 1000 分类输出，重构为 3 分类
model.to(device)

# 7. 损失函数与优化器核心配置
loss_fn = nn.CrossEntropyLoss().to(device)      # 多分类标配：交叉熵损失函数
learning_rate = 1e-3 
optimizer = torch.optim.SGD(model.parameters(), lr=learning_rate) # 选用强健的 SGD 优化器

# 8. 深度训练全链路四步大循环
epoch = 100 #循环次数,当初借用同学5080的显卡跑了一天一夜才24轮ε=(´ο｀*)))
total_train_step = 0

for i in range(epoch):
    print(f"\n🚀 ======= 第 {i+1}/{epoch} 轮训练全链路启动 =======")

    # ------------------ 步骤一：训练模式开辟 ------------------
    model.train() # 激活 Dropout 与 BatchNorm 权重层
    for data in train_data_loader:
        imgs, targets = data
        imgs, targets = imgs.to(device), targets.to(device)
        
        # 前向传播求出损失值
        outputs = model(imgs)
        loss = loss_fn(outputs, targets)

        # ------------------ 步骤二：反向擦除与梯度更新 ------------------
        optimizer.zero_grad()  # 1. 清空上一批次的残余梯度，防止梯度累加
        loss.backward()        # 2. 逆向计算当前的误差梯度
        optimizer.step()       # 3. 驱动 SGD 沿梯度负方向微调网络内部上亿参数

        total_train_step += 1
        if total_train_step % 20 == 0:
            print(f"    [Train] 累计步数: {total_train_step}, 实时 CrossEntropyLoss: {loss.item():.4f}")

    # ------------------ 步骤三：验证与盲测评估 ------------------
    model.eval() # 锁死权重，关闭不必要的随机变动
    total_test_loss = 0
    total_accuracy = 0
    
    with torch.no_grad(): # 切断梯度图计算，压榨边缘端算力，加速验证
        for data in test_data_loader:
            imgs, targets = data
            imgs, targets = imgs.to(device), targets.to(device)
            outputs = model(imgs)
            loss = loss_fn(outputs, targets)

            total_test_loss += loss.item()
            # 统计当前批次正确预测的个数
            accuracy = (outputs.argmax(1) == targets).sum()
            total_accuracy += accuracy

    # ------------------ 步骤四：评价指标收敛与模型快照保存 ------------------
    print(f"📈 [Eval] 整体测试集总损耗 Loss: {total_test_loss:.4f}")
    print(f"🎯 [Eval] 整体测试集终验准确率 (Accuracy): {total_accuracy / test_data_size * 100:.2f}%")
    
    # 每一轮训练结束保存快照模型，便于后续进行推理比对
    torch.save(model, f"yanshi_{i}.pth")
    print(f"💾 快照归档成功: yanshi_{i}.pth")

📹 第三步：边缘端实时分拣与硬件控制模块 (main_video.py) ​

训练出来的 .pth 模型，最终需要下放到产线上。该脚本负责利用 OpenCV 抽取视频流，经过 PyTorch 推理出具体的叶片部位后，将电信号下达给树莓派，操作物理引脚拉动舵机分拣。

Python

import time
import cv2
import os
import torch
import torchvision
from torch import nn
from torchvision import transforms
from PIL import Image

# 💡 边缘端树莓派环境可完全解开下方 GPIO 引脚驱动
# import RPi.GPIO as GPIO
# GPIO.setwarnings(False)
# GPIO.setmode(GPIO.BCM)
# GPIO.setup(26, GPIO.OUT)  # 控制上部叶分拣臂
# GPIO.setup(13, GPIO.OUT)  # 控制中部叶分拣臂
# GPIO.setup(19, GPIO.OUT)  # 控制下部叶分拣臂
# pwm = [GPIO.PWM(26, 50), GPIO.PWM(13, 50), GPIO.PWM(19, 50)]
# pwm[0].start(0); pwm[1].start(0); pwm[2].start(0)

def smooth_pwm_duty_cycle(duty_cycle, smoothed_duty_cycle):
    """一阶低通移动平均滤波算法：防止硬件舵机因目标值骤变产生剧烈抖动与机械撞击"""
    smoothing_factor = 0.9  
    return smoothing_factor * smoothed_duty_cycle + (1 - smoothing_factor) * duty_cycle

def Kong_zhi(x, y, z, i):
    """驱动物理硬件运动：利用循环将离散角度转化为连续阻尼滑行"""
    print(f'🤖 物联网联动：正在平滑驱动第 {i} 号硬件分拣舵机')
    # 步进：摆动角度从 90度 渐变到 45度
    for angle in range(x, y, -1):
        duty_cycle = (angle / 18) + 2.5 # 标准 50Hz 偏转占空比映射公式
        z = smooth_pwm_duty_cycle(duty_cycle, z)
        # pwm[i].ChangeDutyCycle(z)
        time.sleep(0.005)

    # 复位：摆动角度从 45度 摆回 90度
    for angle in range(y, x):
        duty_cycle = (angle / 18) + 2.5
        z = smooth_pwm_duty_cycle(duty_cycle, z)
        # pwm[i].ChangeDutyCycle(z)
        time.sleep(0.005)
    # pwm[i].ChangeDutyCycle(0) # 归零卸载电流，防止舵机持续发热啸叫

transforms_pipeline = transforms.Compose([
    transforms.Resize(256),
    transforms.CenterCrop(224),
    transforms.ColorJitter(contrast=0.5), 
    transforms.ToTensor()
])

def decide(f):
    """核心二次决策机制：结合独热预测得分，追加非线性保护阈值，过滤由于工业阴影带来的环境噪声"""
    num = f[3] # 基础 argmax 索引位置
    if num == 2:
        return 2 # 明确断定为上部叶
    if num == 1:
        if f[2] > f[1] or f[2] > 0: # 概率动态比对条件
            return 1 
        else:
            return 0 # 降级容错机制：划归为下部叶
    return 0

if __name__ == "__main__":
    cap = cv2.VideoCapture(1) # 1 驱动外接工业 USB 高速摄像头
    print("📡 工业流烟叶视觉在线分拣系统已就绪...输入 'm' 单帧检测，输入 'q' 安全退出。")
    
    while True:
        ret, frame = cap.read()
        if not ret:
            break
            
        cv2.imshow("Production_Line_Camera", frame)
        k = cv2.waitKey(1)
        
        if k == ord('q'):
            break
            
        if k == ord('m'): # 手动或对射红外传感器触发
            # 将 OpenCV 采集的 BGR 图像通道重新洗牌为符合模型的 RGB 模式
            img = Image.fromarray(cv2.cvtColor(frame, cv2.COLOR_BGR2RGB))
            img = transforms_pipeline(img)
            img = torch.reshape(img, (1, 3, 224, 224)) # 升维变换扩充 Batch 维度为 1
            
            # 推理端为了高兼容性，可在 CPU 环境下平滑跑通
            model = torch.load("yanshi_0.pth", map_location=torch.device('cpu'))
            model.eval()
            
            with torch.no_grad():
                output = model(img)
            
            # 解析特征空间
            f = [
                output[:, 0].item(),    # 0 索引：下部叶预测置信度
                output[:, 1].item(),    # 1 索引：中部叶预测置信度
                output[:, 2].item(),    # 2 索引：上部叶预测置信度
                output.argmax(1).item() # 3 索引：概率最大品类的归属索引

📂 2. 验证集批量测压脚本 (test_batch.py) ​

一键跑通某一整个归档文件夹下的全部烟叶图片，输出真实的单项指标成功率，在版本迭代中充当评估利器。

Python

import torch
from torchvision import transforms
from PIL import Image
import os

dir_path = "./dataset2/test/bottom" # 指定盲测下部叶文件夹
files = os.listdir(dir_path)
print(f"📊 压测样本总量: {len(files)}")

transforms_pipeline = transforms.Compose([
    transforms.Resize(256), transforms.CenterCrop(224), transforms.ToTensor()
])

device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model = torch.load("yanshi_0.pth").to(device)

total_correct = 0

for file in files:
    img = Image.open(os.path.join(dir_path, file))
    img = transforms_pipeline(img)
    img = torch.reshape(img, (1, 3, 224, 224)).to(device)
    
    model.eval()
    with torch.no_grad():
        output = model(img)
    
    # 若模型输出索引成功预测为 0 (代表bottom)，则正确计数累加
    if output.argmax(1).item() == 0:
        total_correct += 1
    print(f"文件: {file} -> 预测归属索引: {output.argmax(1).item()}")

print("=========================================")
print(f"🏆 压测完结！成功拦截并正确分拣数量: {total_correct}")
print(f"🎯 该品类单项分拣成功率: {total_correct / len(files) * 100:.2f}%")

🛠️ 关于网络骨干 VGG19 的深层解析 ​

1. 为什么 25GB 的大任务敢选择 VGG19？ ​

VGG19 的核心灵魂是通过连续堆叠 3×3 的密集小卷积核，平替早期的 7×7 大卷积核。 2 个 3×3 卷积层的串联，其物理感受野（Receptive Field）等价于一个 5×5 的大卷积；而 3 个 3×3 的堆叠则可以直接等价于 1 个 7×7。

  • 参数大瘦身：3 个 3×3 的参数开销是 3×(3×3)=27，而 1 个 7×7 却高达 49。参数量直降 44.8%，在面对 25GB 级别数据洪峰时，能极大平抑显存占用并提升计算增益。
  • 更卓越的非线性拟合力：由于层数变多，中间可连续加入 3 个非线性激活函数（ReLU），这赋予了 VGG19 对烟叶表面的绒毛、细小撕裂残伤及边缘轮廓极度敏感捕捉的能力。

2. 下一步优化伏笔 ​

虽然 VGG19 是极佳的 Baseline 基准线，但其由于包含长达 4096 节点的全连接层，导致导出的 .pth 模型文件很容易逼近 500MB+。这在树莓派等边缘算力卡口的硬件上容易引发毫秒级延时。

💡 调优进化方向：在大数据量下完成首轮特征榨取后，后续可以尝试使用 VGG19 的权重作为教师网络，通过知识蒸馏（Knowledge Distillation）技术，将学到的分类经验无缝传授给轻量化网络 MobileNetV3 或 ShuffleNetV2，或者直接通过 torch.onnx.export 编译出量化后的 ONNX 格式模型，让传送带上的分拣臂反应速度再翻数倍！
