## 一、 云服务器基础与安全组构建

### 1. 实例环境

* **操作系统**：`CentOS 7.9 64位 Linux`
* **管理鉴权**：
  * 默认管理员：`root`
  * 实例初始密码：`cyx.869466424`（请及时通过 `passwd` 命令进行二次加固）
* **终端连接工具**：
  * `Xshell`：基于 SSH 协议的 Linux 终端连接利器。
  * `Xftp`：基于 SFTP 协议的高效文件传输通道。

### 2. 安全组防火墙开放策略
为了保证内外网服务的连通性，必须在阿里云控制台实例安全组中显式放行以下基础设施及自定义调试端口：
* **80**：HTTP 标准网络入口
* **3306**：MySQL 默认容器访问端口
* **5173**：Vue 局部预览/常规宿主端口
* **8084**：Spring Boot 后端 API 核心通信端口

### 3. 运维常用系统级监控命令
```bash
free -h           # 实时查看系统运行内存 (RAM) 占用
df -h             # 查看磁盘存储空间分发与剩余
top               # 激活系统动态进程管理器（按 q 退出）

# ----------------- 端口与进程生命周期排查 -----------------
# 如果运行 lsof 提示 "command not found"，代表系统未预装，需手动补充安装：
sudo yum install lsof -y

# 精准排查特定后端端口（如 8084）被哪个进程 ID (PID) 占用
sudo lsof -i :8084

# 检查本地防火墙状态及监听规则，确保 8084 端口已暴露并处于 LISTEN 状态
sudo netstat -tuln | grep 8084

# 强行释放/杀死异常进程
kill <PID>        # 优雅终止进程
kill -9 <PID>     # 强行杀死进程（慎用，防止产生脏数据）
```

## 二、 基础运行环境：JDK 17 手工安装与变量激活

> 官方二进制包下载：[Oracle Java SE 17 Downloads](https://www.oracle.com/cn/java/technologies/downloads/#java17)

### 1. 安装步骤

```Bash
# 1. 解压安装包
tar -zxvf jdk-17.0.10_linux-x64_bin.tar.gz

# 2. 规范化命名并迁移至系统本地服务目录
mv jdk-17.0.10 jdk17
mv jdk17 /usr/local/
cd /usr/local/

# 3. 编辑全局环境变量配置文件
vim /etc/profile
```

### 2. 环境变量注入

进入 `vim` 后，键入 `i` 开启编辑模式，在文件底部追加以下三行（**修复了原配置中路径符号拼写错误**）：

Plaintext

```
export JAVA_HOME=/usr/local/jdk17
export CLASSPATH=.:$CLASSPATH:$JAVA_HOME/lib
export PATH=$PATH:$JAVA_HOME/bin
```

按 `Esc` 退出编辑模式，输入 `:wq` 保存并退出。

### 3. 配置刷新与核验

```Bash
cat /etc/profile    # 打印查看是否更改成功
source /etc/profile # 强行刷新当前 shell 环境使变量立即生效
java -version       # 检查是否成功输出 JDK 17 版本信息
```

## 三、 容器化基础设施：Docker 引擎及加速器落地

> 参考教程：[菜鸟教程 - CentOS Docker 安装](https://www.runoob.com/docker/centos-docker-install.html)

### 1. 彻底清理系统旧版冲突组件

```Bash
sudo yum remove docker \
                  docker-client \
                  docker-client-latest \
                  docker-common \
                  docker-latest \
                  docker-latest-logrotate \
                  docker-logrotate \
                  docker-engine
```

### 2. 安装底层依赖包与换源

```Bash
# 安装必要工具包
sudo yum install -y yum-utils device-mapper-persistent-data lvm2

# 添加阿里云官方高可用 Docker 镜像源
sudo yum-config-manager \
    --add-repo \
    [https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo](https://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo)
```

### 3. 安装并拉起核心引擎

```Bash
# 安装 Docker 社区版、命令行工具及容器运行时
sudo yum install docker-ce docker-ce-cli containerd.io docker-compose-plugin -y

# 启动守护进程并配置开机自启
sudo systemctl start docker
systemctl enable docker

# 验证安装是否成功
docker version
```

### 4. 进阶用户权限组与基础运维

为了避免每次执行容器命令都需要敲 `sudo`，建议将当前用户注入 Docker 专属用户组：

```Bash
sudo usermod -aG docker $USER
```

常用日常运维指令：

```Bash
docker images     # 查看本地存储的所有镜像文件
docker ps         # 查看当前正在运行的容器进程
docker ps -a      # 深度排查所有历史容器（含已停止运转的容器）
```

### 5. 国内镜像加速器注入（防止 pull 超时）

```Bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
    "registry-mirrors": ["[https://3n4m4jry.mirror.aliyuncs.com](https://3n4m4jry.mirror.aliyuncs.com)"]
}
```

### 6. Docker 引擎完全卸载（备忘）

```Bash

```

## 四、 隔离运行：Docker 多环境 MySQL 8.0 部署实战

### 1. 拉取官方镜像

```Bash
docker pull mysql:8.0
```

### 2. 容器多阶段/多端口部署路径演进（按需选择）

- **路径一：标准的生产数据卷挂载启动（推荐，数据防丢失）**

```Bash
docker run -p 3306:3306 --name mysql \
-v /mydata/mysql/log:/var/log/mysql \
-v /mydata/mysql/data:/var/lib/mysql \
-v /mydata/mysql/conf:/etc/mysql \
-e MYSQL_ROOT_PASSWORD=root \
-d mysql:8.0
```

- **路径二：无挂载的轻量级单端口测试启动（仅限开发）**

```Bash
docker run -p 3309:3306 --name mysql -e MYSQL_ROOT_PASSWORD=root -d mysql:8.0
```

- **路径三：单机多实例部署 - 多端口隔离挂载（适合单服务器多项目测试）**

```Bash
docker run -p 3309:3306 --name mysql3309 \
-v /mydata/mysql3309/log:/var/log/mysql \
-v /mydata/mysql3309/data:/var/lib/mysql \
-v /mydata/mysql3309/conf:/etc/mysql \
-e MYSQL_ROOT_PASSWORD=root \
-d mysql:8.0
```

- **路径四：当前环境采用的快捷启动命令**

```Bash
docker run -p 3306:3306 --name mysql -e MYSQL_ROOT_PASSWORD=cyx -d mysql:8.0

# 基础控制
docker start mysql # 唤醒并运行该容器实例
```

### 3. 破局高级网络鉴权与外部直连

使用外网工具（如 Navicat）远程拷贝或同步结构时，由于高版本安全校验，需要登录容器内进行数据库授权调优：

```Bash
# 1. 沉浸式切入容器内终端
docker exec -it mysql /bin/bash

# 2. 键入数据库凭证登录
mysql -uroot -pcyx123

# 3. [SQL 执行区]：解除 host 限制并破除高版本加密规则
-- 查询现有用户权限分布
SELECT user, host, authentication_string FROM mysql.user;

-- 将 root 账号限制开放至任意主机 (%) 
ALTER USER 'root'@'%' IDENTIFIED WITH mysql_native_password BY 'cyx123';
ALTER USER 'root'@'%' IDENTIFIED BY "cyx123";

-- 全局域超级授权并强行刷新缓存
GRANT ALL PRIVILEGES ON *.* TO 'root'@'%';
FLUSH PRIVILEGES;
```

### 4. 容器状态常规维护

```Bash
docker stop mysql    # 优雅关闭容器
docker start mysql   # 快速拉起容器
docker ps -a         # 全盘核对容器运行健康度
```

## 五、 后端生产蜕变：Spring Boot 项目打包与宿主

> 参考原理：[CSDN 编译架构全景](https://blog.csdn.net/qq_34491508/article/details/91490434)

### 1. 编译前配置平移（避坑死律）

在本地打包前，务必全盘排查 `application.yml` 或 `properties` 配置文件：

1. **连接端点变更**：将所有本地的 `127.0.0.1` 地址转换为**云服务器真实的公网 IP**（如挂载的 Docker 3306）。
2. **静态路径解耦**：将图片上传/本地下载等存储路径，由 Windows 的 `D:\Pictures\...` 替换为 Linux 的规范绝对路径（如 `/var/data/images/`）。

### 2. 编译打包命令

在本地项目根目录下运行 Maven 指令（跳过单测，提升编译速度）：

```Bash
mvn clean package -DskipTests
```

### 3. IDEA 自动化构建（可选配置）

确保 `pom.xml` 中引入了标准的编译核心插件：

```XML
<build>
    <plugins>
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
        </plugin>
    </plugins>
</build>
```

### 4. 云端宿主运行与后台永久守护

将通过 Xftp 上传至服务器的 `tobacco-0.0.1-SNAPSHOT.jar` 包投入运行：

- **本地特定版本调试运行**：

```Bash
# 显式指定服务器高版本 JDK 绝对路径进行隔离运行
/usr/local/jdk17/bin/java -jar tobacco-0.0.1-SNAPSHOT.jar
```

- **企业级无人值守 - 生产环境后台常驻部署**：

```Bash
# 1. 提权
chmod 777 tobacco-0.0.1-SNAPSHOT.jar

# 2. 配合 nohup 和 & 挂起，把标准输出重定向到 output.log，彻底脱离终端挂断影响
nohup java -jar tobacco-0.0.1-SNAPSHOT.jar > output.log 2>&1 &

# 3. 观测运行轨迹与进程清理
ps -ef | grep java       # 排查目前运行的 Java 进程
kill -9 <后端PID>        # 定向清理旧实例完成热更
```

## 六、 前端工程上云：Vue 编译与轻量级宿主

### 1. 网络请求端点对齐 (Axios)

打包前检查前端代码中的代理控制文件（如 `request.js` 或 `vite.config.ts`）。务必将本地联调的 `http://127.0.0.1:8084` 重构指向为**云服务器的公网 IP 或者是已被解析的域名端点**。

### 2. package.json 编译脚本优化

在生产环境中，静态强类型检查器 `vue-tsc` 有时会因代码内微小的类型警示强行中断编译。因此生产线打包时建议绕过校验，直接使用 Vite 底层的高速构建：

```JSON
"scripts": {
    "test": "vite build",
    "build": "vue-tsc && vite build"
}
```

**生产打包推荐执行**：

```Bash
npm run test   # 绕过 vue-tsc 校验，直接产出纯净的 dist 静态集
```

### 3. 云端 Node.js 宿主环境手工搭建

利用 Xftp 将本地编译出的 `dist` 压缩包上传到云端自定义根目录并解压：

```Bash
# 1. 注入官方 Node 16.x LTS 长效支持版包源
curl -fsSL [https://rpm.nodesource.com/setup_16.x](https://rpm.nodesource.com/setup_16.x) | sudo bash -

# 2. 手工安装 Node.js 与伴生包管理器 npm
sudo yum install -y nodejs

# 3. 版本核验
node -v
npm -v
```

### 4. 轻量级静态网页服务器挂载与后台挂起

为了不改动大型 Nginx，可以利用 Node 原生生态的 `http-server` 快速完成对 `dist` 资源的生产宿主：

```Bash
# 1. 全局脚手架安装静态服务器
npm install -g http-server

# 2. 前台常规测试启动（占用当前终端）
http-server -p 5173

# 3. 生产级无人值守后台挂起（关闭终端依然可用）
nohup http-server -p 5173 > /dev/null 2>&1 &
```

## 🚀 终极架构设想：全栈一键自动化部署脚本（.sh）

为了免除上述繁琐的逐个命令配置，未来项目可以规划采用自动化工程，通过把全量包、SQL脚本和以下 `.sh` 脚本统包，完成全自动上云：

Bash

```bash
#!/bin/bash
# 自动化一键部署概念雏形
echo "======= 正在停止并清理旧服务 ======="
kill -9 $(lsof -t -i:8084) 2>/dev/null
kill -9 $(lsof -t -i:5173) 2>/dev/null

echo "======= 正在启动后端 Spring Boot ======="
nohup java -jar /usr/local/app/tobacco-0.0.1-SNAPSHOT.jar > /dev/null 2>&1 &

echo "======= 正在拉起前端 Vue 页面 ======="
cd /usr/local/app/dist
nohup http-server -p 5173 > /dev/null 2>&1 &

echo "======= 🎉 全栈服务已完成自动化复苏！======="
---
```
