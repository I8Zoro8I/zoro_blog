# Apache Spark - 大规模数据分析的统一引擎



Apache Spark 是一个快速的，通用的集群计算系统。它对 Java，Scala，Python 和 R 提供了的高层 API，并有一个经优化的支持通用执行图计算的引擎。它还支持一组丰富的高级工具，包括用于 SQL 和结构化数据处理的 [Spark SQL](https://geekdaxue.co/read/spark-doc-zh/$docs-sql-programming-guide.html)，用于机器学习的 [MLlib](https://geekdaxue.co/read/spark-doc-zh/$docs-ml-guide.html)，用于图计算的 [GraphX](https://geekdaxue.co/read/spark-doc-zh/$docs-graphx-programming-guide.html) 和 [Spark Streaming](https://geekdaxue.co/read/spark-doc-zh/$docs-streaming-programming-guide.html)。

# PySpark

PySpark 是 Apache Spark 的 Python API。它使您能够在分布式环境中使用 Python 执行大规模的实时数据处理。它还提供了一个 PySpark shell，用于交互式地分析数据。

PySpark 结合了 Python 的易学易用性与 Apache Spark 的强大功能，使熟悉 Python 的用户能够处理和分析任意规模的数据。

PySpark 支持 Spark 的所有功能，如 Spark SQL、DataFrame、结构化流（Structured Streaming）、机器学习（MLlib）、流水线（Pipelines）和 Spark Core。

Spark Connect 是 Apache Spark 中的一种客户端-服务器架构，它允许任何应用程序远程连接到 Spark 集群。PySpark 提供了 Spark Connect 服务器的客户端，使得 Spark 可以作为一种服务来使用。

**Spark SQL 和 DataFrame**

Spark SQL 是 Apache Spark 用于处理结构化数据的模块。它允许您将 SQL 查询与 Spark 程序无缝结合。利用 PySpark DataFrame，您可以使用 Python 和 SQL 高效地读取、写入、转换和分析数据。无论您使用 Python 还是 SQL，底层都会使用相同的执行引擎，因此您始终能充分发挥 Spark 的强大性能。

**Spark 上的 Pandas API**

Spark 上的 Pandas API 允许您通过跨多个节点进行分布式运行，将 pandas 工作负载扩展到任意规模。如果您已经熟悉 pandas 并希望利用 Spark 进行大数据处理，Spark 上的 pandas API 可以让您立即提高工作效率，并在无需修改代码的情况下迁移应用程序。您可以拥有一套既适用于 pandas（测试、较小数据集）又适用于 Spark（生产环境、分布式数据集）的代码库，并且可以轻松且无开销地在 pandas API 和 Spark 上的 Pandas API 之间切换。

Spark 上的 Pandas API 旨在简化从 pandas 到 Spark 的过渡，但如果您是 Spark 的新手或正在决定使用哪种 API，我们建议使用 PySpark（请参阅 [Spark SQL 和 DataFrames](https://spark.apache.ac.cn/docs/latest/api/python/index.html#index-page-spark-sql-and-dataframes)）。

- [快速入门：Spark 上的 Pandas API](https://spark.apache.ac.cn/docs/latest/api/python/getting_started/quickstart_ps.html)
- [在线 Notebook：Spark 上的 pandas API](https://mybinder.org/v2/gh/apache/spark/c0690c763ba?filepath=python%2Fdocs%2Fsource%2Fgetting_started%2Fquickstart_ps.ipynb)
- [Spark 上的 Pandas API 参考](https://spark.apache.ac.cn/docs/latest/api/python/reference/pyspark.pandas/index.html)

**Structured Streaming**

结构化流（Structured Streaming）是一个基于 Spark SQL 引擎构建的可扩展且容错的流处理引擎。您可以像编写静态数据的批处理计算一样编写流计算。Spark SQL 引擎将负责增量且持续地运行它，并在流数据不断到达时更新最终结果。

- [Structured Streaming 编程指南](https://spark.apache.ac.cn/docs/latest/structured-streaming-programming-guide.html)
- [结构化流（Structured Streaming）API 参考](https://spark.apache.ac.cn/docs/latest/api/python/reference/pyspark.ss/index.html)

**机器学习 (MLlib)**

MLlib 构建于 Spark 之上，是一个可扩展的机器学习库，提供了一套统一的高级 API，帮助用户创建和调整实用的机器学习流水线。

- [机器学习库 (MLlib) 编程指南](https://spark.apache.ac.cn/docs/latest/ml-guide.html)
- [机器学习 (MLlib) API 参考](https://spark.apache.ac.cn/docs/latest/api/python/reference/pyspark.ml.html)

**声明式流水线 (Declarative Pipelines)**

Spark 声明式流水线 (SDP) 是一个用于在 Spark 上构建可靠、可维护和可测试数据流水线的声明式框架。SDP 通过允许您专注于应用于数据的转换，而不是流水线执行的细节，从而简化了 ETL 开发。

- [流水线 (Pipelines) API 参考](https://spark.apache.ac.cn/docs/latest/api/python/reference/pyspark.pipelines.html)

**Spark Core 和 RDD**

Spark Core 是 Spark 平台底层的通用执行引擎，所有其他功能都构建在其之上。它提供了 RDD（弹性分布式数据集）和内存计算能力。

请注意，RDD API 属于低级 API，使用起来较为困难，且无法享受到 Spark 自动查询优化功能带来的好处。我们建议使用 DataFrame（请参阅上文 [Spark SQL 和 DataFrames](https://spark.apache.ac.cn/docs/latest/api/python/index.html#index-page-spark-sql-and-dataframes)）而非 RDD，因为它能让您更轻松地表达意图，并让 Spark 自动为您构建最高效的查询。

- [Spark Core API 参考](https://spark.apache.ac.cn/docs/latest/api/python/reference/pyspark.html)

**Spark Streaming (旧版)**

Spark Streaming 是核心 Spark API 的一个扩展，支持对实时数据流进行可扩展、高吞吐、容错的流处理。

请注意，Spark Streaming 是 Spark 上一代的流处理引擎。它是一个旧版项目，不再更新。Spark 中有一个更新、更易于使用的流处理引擎，称为 [结构化流 (Structured Streaming)](https://spark.apache.ac.cn/docs/latest/api/python/index.html#index-page-structured-streaming)，您应该在流处理应用程序和流水线中使用它。

- [Spark Streaming 编程指南 (旧版)](https://spark.apache.ac.cn/docs/latest/streaming-programming-guide.html)
- [Spark Streaming API 参考 (旧版)](https://spark.apache.ac.cn/docs/latest/api/python/reference/pyspark.streaming.html)

# 安装

PySpark 包含在 [Apache Spark 官网](https://spark.apache.ac.cn/downloads.html)提供的 Spark 官方版本中。对于 Python 用户，PySpark 还提供了从 PyPI 通过 `pip` 安装的方式。这通常用于本地使用，或作为连接到集群的客户端，而不是自行搭建集群。

本页包含使用 pip、Conda、手动下载以及从源代码构建来安装 PySpark 的说明。

## 支持的 Python 版本

Python 3.10 及以上版本。

## 使用 PyPI

使用 [PyPI (pyspark)](https://pypi.ac.cn/project/pyspark/) 安装 PySpark 的方法如下

```
pip install pyspark
```

如果您想为特定组件安装额外的依赖项，可以按照以下方式安装

```
# Spark SQL
pip install pyspark[sql]
# pandas API on Spark
pip install pyspark[pandas_on_spark] plotly  # to plot your data, you can install plotly together.
# Spark Connect
pip install pyspark[connect]
```



有关额外依赖项的更多详细信息，请参阅 [可选依赖项](https://spark.apache.ac.cn/docs/latest/api/python/getting_started/install.html#optional-dependencies)。

对于带有或不带特定 Hadoop 版本的 PySpark，可以使用 `PYSPARK_HADOOP_VERSION` 环境变量进行安装，如下所示

```
PYSPARK_HADOOP_VERSION=3 pip install pyspark
```



默认分发版使用 Hadoop 3.3 和 Hive 2.3。如果用户指定了不同版本的 Hadoop，pip 安装程序会自动下载该版本并在 PySpark 中使用。根据网络情况和所选镜像源，下载可能需要一些时间。可以设置 `PYSPARK_RELEASE_MIRROR` 手动选择镜像源以加快下载速度。

```
PYSPARK_RELEASE_MIRROR=http://mirror.apache-kr.org PYSPARK_HADOOP_VERSION=3 pip install
```



建议在 `pip` 中使用 `-v` 选项以跟踪安装和下载状态。

```
PYSPARK_HADOOP_VERSION=3 pip install pyspark -v
```



`PYSPARK_HADOOP_VERSION` 支持的值为

- `without`：使用用户提供的 Apache Hadoop 预构建的 Spark
- `3`：为 Apache Hadoop 3.3 及更高版本预构建的 Spark（默认）

请注意，这种带有/不带有特定 Hadoop 版本的 PySpark 安装方式尚处于实验阶段。它可能会在次要版本发布之间发生变化或被移除。

### 将 Spark Connect 设置为默认

如果您想将 Spark Connect 设置为默认，可以通过 [PyPI (pyspark-connect)](https://pypi.ac.cn/project/pyspark-connect/) 安装一个额外的库。执行以下命令

```
pip install pyspark-connect
```



这将自动安装 `pyspark` 库以及 Spark Connect 所需的依赖项。如果您想自定义 `pyspark`，则需要提前按照上述说明安装 `pyspark`。

该软件包同时支持：- `spark.master` (`--master`)（使用本地运行的 Spark Connect 服务器）- `spark.remote` (`--remote`)（包括本地集群，例如 `local[*]`，以及连接 URI，如 `sc://`）。

有关如何使用它的信息，请参阅 [快速入门：Spark Connect](https://spark.apache.ac.cn/docs/latest/api/python/getting_started/quickstart_connect.html)。

### Python Spark Connect 客户端

Python Spark Connect 客户端是一个纯 Python 库，不依赖于环境中任何非 Python 的依赖项（如 jar 包和 JRE）。要通过 [PyPI (pyspark-client)](https://pypi.ac.cn/project/pyspark-client/) 安装 Python Spark Connect 客户端，请执行以下命令

```
pip install pyspark-client
```



该软件包仅支持带有连接 URI 的 spark.remote，例如 `sc://`。有关如何使用它的信息，请参阅 [快速入门：Spark Connect](https://spark.apache.ac.cn/docs/latest/api/python/getting_started/quickstart_connect.html)。

## 使用 Conda

Conda 是一个开源的包管理和环境管理系统（由 [Anaconda](https://anaconda.net.cn/) 开发），最好通过 [Miniconda](https://docs.conda.org.cn/en/latest/miniconda.html) 或 [Miniforge](https://github.com/conda-forge/miniforge/) 安装。该工具既跨平台又与语言无关，在实践中，conda 可以替代 [pip](https://pip.pypa.org.cn/en/latest/) 和 [virtualenv](https://virtualenv.pypa.io/en/latest/)。

Conda 使用所谓的“频道”（channels）来分发软件包。除了 Anaconda 自带的默认频道外，最重要的频道是 [conda-forge](https://forge.conda.org.cn/)。这是一个由社区驱动的打包工作，它是最全面且最新的（在大多数情况下也作为 Anaconda 频道的上游）。

要从终端创建新的 conda 环境并激活它，请按如下所示进行操作

```
conda create -n pyspark_env
conda activate pyspark_env
```



激活环境后，使用以下命令安装 pyspark、您选择的 Python 版本，以及您希望与 pyspark 在同一会话中使用的其他软件包（您也可以分多个步骤安装）。

```
conda install -c conda-forge pyspark  # can also add "python=3.10 some_package [etc.]" here
```



请注意，[Conda 版 PySpark](https://anaconda.org/conda-forge/pyspark) 由社区单独维护；虽然新版本通常会很快打包，但 conda(-forge) 的可用性并不直接与 PySpark 的发布周期同步。

虽然在 conda 环境中使用 pip 在技术上是可行的（使用与[上述](https://spark.apache.ac.cn/docs/latest/api/python/getting_started/install.html#using-pypi)相同的命令），但这种做法是[不推荐的](https://anaconda.net.cn/blog/using-pip-in-a-conda-environment/)，因为 pip 无法与 conda 互操作。

有关有用 conda 命令的简要总结，请参阅他们的 [速查表](https://docs.conda.org.cn/projects/conda/en/latest/user-guide/cheatsheet.html)。

## 手动下载

PySpark 包含在 [Apache Spark 网站](https://spark.apache.ac.cn/downloads.html)提供的分发版中。您可以从该网站下载所需的分发版。下载完成后，将 tar 文件解压到您想要安装 Spark 的目录中，例如，如下所示

```
tar xzvf spark-\ |release|\-bin-hadoop3.tgz
```



确保 `SPARK_HOME` 环境变量指向解压 tar 文件的目录。更新 `PYTHONPATH` 环境变量，以便它能找到 `SPARK_HOME/python/lib` 下的 PySpark 和 Py4J。实现此目的的一个示例如下所示

```
cd spark-\ |release|\-bin-hadoop3
export SPARK_HOME=`pwd`
export PYTHONPATH=$(ZIPS=("$SPARK_HOME"/python/lib/*.zip); IFS=:; echo "${ZIPS[*]}"):$PYTHONPATH
```

## 依赖项

### 必需依赖项

PySpark 需要以下依赖项。

| 包   | 支持的版本 | 注意              |
| ---- | ---------- | ----------------- |
| py4j | >=0.10.9.9 | 与 JVM 交互所必需 |

增强功能但未包含在安装包中的额外库

- **memory-profiler**：用于 PySpark UDF 内存分析，`spark.profile.show(...)` 和 `spark.sql.pyspark.udf.profiler`。
- **plotly**：用于 PySpark 绘图，`DataFrame.plot`。

注意：PySpark 需要 Java 17 或更高版本，且必须正确设置 `JAVA_HOME`。请参考 [下载](https://spark.apache.ac.cn/docs/4.1.1/#downloading) 部分。



### 可选依赖项

PySpark 有几个可选依赖项，用于增强特定模块的功能。这些依赖项仅在特定功能需要时才需要，对于 PySpark 的基本功能不是必需的。如果未安装这些可选依赖项，PySpark 的基本操作将正常运行，但当您尝试使用需要这些依赖项的功能时，会引发 `ImportError`。

#### Spark Connect

可通过 `pip install "pyspark[connect]"` 安装。

| 包                       | 支持的版本 | 注意               |
| ------------------------ | ---------- | ------------------ |
| pandas                   | >=2.2.0    | Spark Connect 所需 |
| pyarrow                  | >=15.0.0   | Spark Connect 所需 |
| grpcio                   | >=1.76.0   | Spark Connect 所需 |
| grpcio-status            | >=1.76.0   | Spark Connect 所需 |
| googleapis-common-protos | >=1.71.0   | Spark Connect 所需 |
| zstandard                | >=0.25.0   | Spark Connect 所需 |
| graphviz                 | >=0.20     | Spark Connect 可选 |

#### Spark SQL

可通过 `pip install "pyspark[sql]"` 安装。

| 包      | 支持的版本 | 注意           |
| ------- | ---------- | -------------- |
| pandas  | >=2.2.0    | Spark SQL 所需 |
| pyarrow | >=15.0.0   | Spark SQL 所需 |

增强功能但未包含在安装包中的额外库

- **flameprof**：提供用于 UDF 性能分析的默认渲染器。

#### Spark 上的 Pandas API

可通过 `pip install "pyspark[pandas_on_spark]"` 安装。

| 包      | 支持的版本 | 注意                       |
| ------- | ---------- | -------------------------- |
| pandas  | >=2.2.0    | Spark 上的 Pandas API 所需 |
| pyarrow | >=15.0.0   | Spark 上的 Pandas API 所需 |

增强功能但未包含在安装包中的额外库

- **mlflow**：`pyspark.pandas.mlflow` 所需。
- **plotly**：提供用于可视化的绘图功能。建议优先使用 **plotly** 而非 **matplotlib**。
- **matplotlib**：提供用于可视化的绘图功能。默认值为 **plotly**。

#### 基于 DataFrame 的 MLlib API

可通过 `pip install "pyspark[ml]"` 安装。

| 包    | 支持的版本 | 注意                             |
| ----- | ---------- | -------------------------------- |
| numpy | >=1.22     | 基于 DataFrame 的 MLlib API 所需 |

增强功能但未包含在安装包中的额外库

- **scipy**：SciPy 集成所需。
- **scikit-learn**：实现机器学习算法所需。
- **torch**：机器学习模型训练所需。
- **torchvision**：支持图像和视频处理所需。
- **torcheval**：促进模型评估指标所需。
- **deepspeed**：提供高性能模型训练优化所需。可在非 Darwin 系统上安装。

#### MLlib

可通过 `pip install "pyspark[mllib]"` 安装。

| 包    | 支持的版本 | 注意       |
| ----- | ---------- | ---------- |
| numpy | >=1.22     | MLlib 所需 |

#### 声明式管道

可通过 `pip install "pyspark[pipelines]"` 安装。包含 Spark SQL 和 Spark Connect 的所有依赖项，因为声明式管道是建立在这两者之上的。

| 包                       | 支持的版本 | 注意                            |
| ------------------------ | ---------- | ------------------------------- |
| pandas                   | >=2.2.0    | Spark Connect 和 Spark SQL 所需 |
| pyarrow                  | >=15.0.0   | Spark Connect 和 Spark SQL 所需 |
| grpcio                   | >=1.76.0   | Spark Connect 所需              |
| grpcio-status            | >=1.76.0   | Spark Connect 所需              |
| googleapis-common-protos | >=1.71.0   | Spark Connect 所需              |
| zstandard                | >=0.25.0   | Spark Connect 所需              |
| pyyaml                   | >=3.11     | spark-pipelines 命令行界面所需  |
| graphviz                 | >=0.20     | Spark Connect 可选              |
