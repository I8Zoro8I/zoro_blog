## 第一阶段：基础能力打底（1-2 个月）

目标：搞定岗位要求里的数学、Python 和编程基础，为后面学 AI 铺路。

### 1. 数学基础（AI 的底层逻辑）

- 必学内容

  - 线性代数：矩阵 / 向量运算、特征值、奇异值分解（SVD）、PCA 降维
- 概率论与数理统计：概率分布、贝叶斯定理、期望 / 方差、假设检验
  - 微积分：偏导数、梯度、链式法则（理解神经网络反向传播必备）
  
  

- **学习方式**：看 MIT 18.06 线性代数公开课 +《统计学习方法》的数学基础部分，配合 B 站李永乐老师的视频辅助理解。

### 2. Python 与编程基础（岗位要求 5）

- 核心技能

  ：

  - 熟练 Python 语法、数据结构（列表 / 字典 / 元组 / 集合）、函数、类与面向对象
  - 掌握 Numpy（矩阵运算）、Pandas（数据处理）、Matplotlib/Seaborn（数据可视化）
  - 掌握基础数据处理流程：数据清洗、特征工程、数据集划分

  

- **学习方式**：跟着 Python 实战教程敲代码，完成 3-5 个数据处理小项目（比如用户行为分析、数据可视化报告）。

------

## 第二阶段：机器学习基础（2-3 个月）

目标：掌握岗位要求 4 里的常用算法，能独立完成传统机器学习项目。

### 1. 核心算法学习

按「原理 + 代码实现 + 项目实战」的顺序学：

- 基础算法：线性回归、逻辑回归、决策树、随机森林、XGBoost/LightGBM
- 无监督算法：聚类（K-Means）、降维（PCA）
- 进阶内容：模型评估（准确率、召回率、F1、AUC）、过拟合与正则化、交叉验证
- **重点工具**：Scikit-learn 库，所有算法都要亲手跑一遍示例代码。

### 2. 实战项目（必须动手！）

- 项目 1：房价预测（回归问题，练线性回归 + 随机森林）
- 项目 2：客户流失预测（分类问题，练逻辑回归 + XGBoost）
- 项目 3：用户分群（无监督，练 K-Means 聚类）
- 项目要求：写出完整的代码、分析报告，理解每一步的调参逻辑。

------

## 第三阶段：深度学习框架与模型（3-4 个月）

目标：搞定岗位要求 3 的框架，掌握神经网络相关技术，匹配模型开发与优化需求。

### 1. 深度学习框架（岗位要求 3）

- 首选 PyTorch

  （工业界和面试更常用），次选 TensorFlow

  - 核心学习：张量操作、自动求导、神经网络搭建（nn.Module）、数据集加载（DataLoader）
  - 基础模型实现：全连接神经网络、CNN（卷积神经网络）、RNN/LSTM

  

- **补充 Keras/TensorFlow**：了解基础用法，能看懂和复现简单模型即可。

### 2. 进阶深度学习内容

- 重点学习：CNN 图像分类、RNN/Transformer 文本处理（NLP 基础）、模型训练与调优（学习率、优化器、batch size）

- 项目实战：

  - 项目 1：手写数字识别（MNIST，练基础 CNN）
  - 项目 2：文本情感分类（练 LSTM / 简单 Transformer）
  - 项目 3：简单模型优化（比如剪枝、量化，理解模型性能提升方法）

  

------

## 第四阶段：AI 工程化与加分项（2-3 个月）

目标：覆盖岗位职责里的数据集管理、模型训练，以及要求 6 的大数据工具，提升项目完整度。

### 1. 数据集与模型工程（对应岗位职责 3）

- 数据标注工具：LabelImg（图像）、LabelStudio（多模态），理解标注规范与数据清洗
- 模型训练流程：训练集 / 验证集 / 测试集划分、训练日志记录（TensorBoard）、模型保存与部署基础（ONNX、TorchScript）
- 提示词工程：学习大模型基础提示词技巧（少样本提示、思维链），写一份给非技术人员的 AI 工具使用文档（对应岗位职责 4）

### 2. 大数据加分项（岗位要求 6）

- 基础学习：Hadoop/Spark 的核心概念（分布式存储、分布式计算），了解 Spark SQL 和 PySpark 的基础用法
- 实战小任务：用 PySpark 处理一份大数据集（比如百万级用户日志），做简单的统计分析，理解分布式处理的优势。

------

## 第五阶段：项目整合与简历准备（持续进行）

目标：把前面的内容串起来，形成能写进简历的项目经验，匹配岗位要求 2 的 1 年经验 / 项目经验。

### 1. 打造 2-3 个完整的 AI 项目

- 项目标准：

  从数据到部署的全流程

  ，每个项目都要有：

  - 问题背景、数据集来源
  - 数据清洗与特征工程过程
  - 模型选型与训练调优（对比不同算法 / 框架的效果）
  - 模型评估与优化结果
  - 完整的代码仓库（GitHub）+ 项目报告 / 演示视频

  

- 示例项目：

  - 端到端的用户画像与流失预测系统（传统 ML+Spark 数据处理）
  - 基于 PyTorch 的图像分类模型 + 简单部署（比如用 FastAPI 做 API 服务）

  

### 2. 简历与面试准备

- 把项目经验和岗位 JD 对应起来：

  - 岗位职责 1：写清楚「参与 AI 产品设计与开发，负责 XX 模型的实现与优化，提升准确率 X%」
  - 岗位职责 3：写清楚「负责数据集管理与标注，处理 XX 万条数据，建立数据清洗规范」
  - 岗位职责 4：写清楚「撰写技术文档与提示词模板，帮助团队落地 AI 工具，提升效率 X%」

  

- 面试准备：整理常见算法原理、模型调优经验、项目中的难点与解决方案。

------

## 学习节奏与小建议

- 总周期：按每天 2-3 小时学习，大概**8-12 个月**能走完这个路线，匹配岗位基础要求。
- 关键原则：**边学边练，拒绝纯理论**。每个知识点都要配合代码和小项目，不然很容易学了就忘。
- 优先顺序：Python → 数学基础 → 传统机器学习 → PyTorch 深度学习 → 工程化与项目整合，不要跳级学习。
